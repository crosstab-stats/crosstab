DuckDB-WASM ↔ WebR bridge spike

Proves the chosen data engine before anything depends on it: load DuckDB-WASM, generate a large synthetic dataset entirely in-engine, push an aggregate/sample down to DuckDB, then get the reduced result into WebR and run real R on it — measuring peak memory and timings throughout. Needs cross-origin isolation (COOP/COEP). Throwaway; not wired into the app.

Synthetic dataset

200 × 500k doubles ≈ 800 MB. Dial back if the runtime OOMs — finding the ceiling is part of the experiment.
